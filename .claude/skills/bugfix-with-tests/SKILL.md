# Bugfix With Tests

## Purpose
バグ修正を最小変更で行い、再発防止テストまで含めて完了するためのワークフロー。

## When to Use
- 既存不具合を修正するとき
- 表示崩れ、日時ズレ、保存処理の不整合などを直すとき
- 原因箇所が複数ファイルにまたがるとき

## Workflow
1. 症状を確認する
2. 原因箇所を特定する
3. 保存時 / 取得時 / 表示時のどこでズレているかを切り分ける
4. 最小変更で修正方針を決める
5. 実装する
6. 再発防止テストを追加する
7. lint / typecheck / test を実行する
8. git add / git commit を行う
9. 修正内容と残課題を報告する

## Guardrails
- 症状だけ見て場当たり的に直さない
- 不必要なリファクタを混ぜない
- 既存 API や UI の責務を崩さない
- バグ修正には必ずテストを追加する
- コミットはバグ修正単位で行う

## Checklist
- 原因は説明できるか
- null ケースは安全か
- 編集時も新規作成時も整合しているか
- テストが追加されているか
- 変更範囲は最小か
