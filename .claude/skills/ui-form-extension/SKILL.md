# UI Form Extension

## Purpose
フォーム項目の追加や編集を、型・DB・UI・表示の整合を保ちながら進めるためのワークフロー。

## When to Use
- application に新しい入力項目を追加するとき
- フォームの入力とカード表示を連動させたいとき
- 軽量な機能追加を行うとき

## Workflow
1. 追加する項目の責務を確認する
2. status など既存フィールドに混ぜるべきでない責務がないか確認する
3. 型定義を更新する
4. DBアクセス層を更新する
5. フォーム入力 UI を更新する
6. カードや一覧の表示を更新する
7. 必要ならフィルタや抽出導線を追加する
8. テストを追加する
9. lint / typecheck / test を実行する
10. git add / git commit を行う

## Guardrails
- 1つのフィールドに複数責務を混ぜない
- application に自然に乗る範囲で最小変更に留める
- 保存先が既存フィールドなら docs の判断に従う
- UI 変更だけ先行して型や保存処理を置き去りにしない

## Checklist
- 型は更新したか
- queries は更新したか
- フォームで入力 / 編集できるか
- 表示側に反映されるか
- テストは追加したか
