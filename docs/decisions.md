# docs/decisions.md

## Purpose
このファイルには、現時点の設計判断を記録する。
不変ルールではなく、将来変更されうる方針をここで管理する。

---

## Current Decisions

### 1. 日時の扱い
- 面談日時や締切日は、ユーザーが入力したローカル時刻として扱う
- UI 上では、入力した日時と同じ見え方になることを優先する
- 実装時は「保存時」「取得時」「表示時」の各段階でタイムゾーンずれが起きないようにする

### 2. 締切日の自動リマインド
- まずは画面表示のみを行う
- 対象は、締切日の表示・近接警告・期限切れ表示
- 通知、メール送信、バックグラウンドジョブはこの時点では扱わない

### 3. 面談メモテンプレ
- 面談後の感想・魅力点・懸念点・次回聞きたいことは、まず memo に含める
- 直ちに構造化カラムへ分離しない
- テンプレ入力は UI で補助し、保存先は既存 memo を使う

### 4. 逆求人イベント記録
- 既存 application とは別概念として扱う
- 通常応募とは異なる接点情報として整理する
- application に無理に押し込まず、必要なら専用エンティティを設ける前提で考える

### 5. メール文面生成
- メール文面は自動生成だけでなく保存も行う
- 生成結果を一時表示で捨てず、履歴として扱える設計を前提にする
- 送信機能を直ちに含めるかどうかは別判断とする

## Near-Term Feature Decisions

### 応募種別分類
- 夏インターン / 本選考 / 内定者インターンは、status とは別軸で扱う
- status に混ぜず、別フィールドとして管理する前提で進める

### Webテスト管理
- Webテスト受験状況は status と分けて管理する
- 「未受験企業の抽出」ができる状態を優先する

---

## Decision Update Rule
- 新しい実装判断をしたら CLAUDE.md ではなくこのファイルを更新する
- 実装詳細ではなく、判断の理由と方針を残す
- 迷ったら「不変か / 可変か」で保存先を決める
